SqlDefrag 2008

Installation Package
Source Code

The project was put underway when the need arose to have non-database "savvy" individuals to rebuild or reorganize indices on a Microsoft SQL 2008 server database.

Rebuilding of an index is based upon the variable upperbound limit for fragmentation while a re-organization of the index is based upon the values between the upperbound and lowerbound fragmentations limits imposed.

The utility also provides the ability to just "Touch" the database. This allows for an initial gauge of what state the database is in to be assessed prior to executing any SQL commands. This was a necessary feature because if any external operations utilize cursors during this maintenance the possibility of conflicts grows.

Initial settings are based from a blog post from Pinal Dave.
